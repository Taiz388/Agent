# 合同生成智能体测试方法（极简界面版）

本测试文档用于验证 `D:\合同生成` 项目的合同生成智能体。新版界面默认不需要左侧导航，也不需要填写大量字段。

本地访问地址：

```text
http://localhost:8501
```

通用操作：

1. 打开网页。
2. 保持默认设置，不打开“高级选项”。
3. 点击“上传合同资料”。
4. 按测试例上传对应文件。
5. 点击“生成合同”。
6. 下载生成的 Word 合同。
7. 如需核对字段，展开“查看识别结果”。

说明：

- “高级选项”只在自动识别不准时使用，通常不用填写。
- “测试样例说明”在网页底部折叠显示。
- 参考资料不上传到 GitHub，测试时使用本机 `D:\合同生成\参考资料` 里的文件。

---

## 测试例 1：标书资料生成合同 - 16#挤压筒换内衬

测试目标：验证系统能够从竞争谈判文件、中选通知书、最终合同样本中生成采购买卖合同。

上传文件所在目录：

```text
D:\合同生成\参考资料\交易生成合同\MLBB（20）24-29616#挤压筒换内衬
```

上传文件：

```text
竞争谈判文件（16#挤压筒换内衬）.docx
中选通知书.docx
MLBB（20）24-29616#挤压筒换内衬.docx
```

页面选择：

```text
不需要选择，保持默认自动识别即可。
```

预期结果：

```text
合同类型：采购买卖合同
合同编号：MLBB（20）24-296
需方：福建省南平铝业股份有限公司
供方：江苏xxx集团股份有限公司
项目/产品：16#挤压筒换内衬
金额：285000
```

通过标准：生成 Word 合同，且合同正文中能看到 `16#挤压筒换内衬`、`江苏xxx集团股份有限公司`、`285000`。

## 测试例 5：意向书生成合同 - 暂无真实资料的兜底测试

测试目标：验证在没有真实意向书样本的情况下，系统仍可根据简短业务描述生成合同初稿。

测试方法：新建一个 Word 或 txt 文件，内容如下：

```text
福建省南平铝业股份有限公司拟向测试供应商有限公司采购铝板产品一批，合同金额暂定100000元，交货地点为需方指定仓库，付款方式为货到验收合格并收到13%增值税专用发票后付款。
```

页面选择：

```text
默认自动识别也可以；如果需要，可展开“高级选项”，资料类型选择“意向书生成合同”。
```

预期结果：

```text
系统生成买卖合同或铝板销售合同初稿。
需方、供方、产品、金额、交货地点、付款方式应尽量被识别。
```

通过标准：生成 Word 合同，合同结构完整，缺失信息可在“查看识别结果”中核对。
